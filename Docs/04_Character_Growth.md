# Character Growth System

## 1. 개요

PoF의 캐릭터 성장 시스템은  
레벨, 경험치, 승급(Promotion)을 중심으로 설계되었습니다.

성장의 핵심 목표는 다음과 같습니다.

- 명확한 성장 루프 제공
- 재료 기반 성장 구조
- 캐릭터별 차별화된 성장 경험
- UI에서 즉시 피드백 제공 (프리뷰 시스템)

---

## 2. 성장 구조

### 기본 구성

캐릭터 성장은 다음 3가지 요소로 구성됩니다.

1. Level (레벨)
2. EXP (경험치)
3. Promotion (승급)

---

### 레벨 시스템

- 경험치를 획득하면 레벨이 상승
- 레벨 상승 시 기본 스탯 증가
- 특정 레벨 구간에서 승급 필요

👉 단순 레벨업이 아니라 **성장 제한 구조 포함**

---

### 승급 시스템

- 일정 레벨 도달 시 승급 필요
- 승급 시:
  - 레벨 제한 해제
  - 스탯 성장 폭 증가
  - 추가 성장 가능 상태 진입

👉 “구간 성장 → 승급 → 다음 구간” 구조

---

## 3. 재료 기반 성장

성장은 아이템(재료)을 통해 진행됩니다.

### 특징

- 캐릭터별 전용 재료 존재
- 재료 선택 → EXP 변환 구조
- 재료 수량에 따라 성장량 변화

---

### LevelUp 방식

1. 재료 선택
2. EXP 계산
3. 레벨 상승 시뮬레이션
4. UI 프리뷰 반영

👉 실제 적용 전에 결과를 미리 확인 가능

---

## 4. 프리뷰 시스템 (중요)

이 시스템은 UX적으로 매우 중요한 부분입니다.

### 동작 방식

- 현재 상태를 복사
- 재료 적용 후 예상 결과 계산
- UI에 “적용 전 / 적용 후” 비교 표시

### 표시 정보

- 예상 레벨
- 예상 스탯 증가
- EXP 변화
- 승급 가능 여부

👉 플레이어가 리스크 없이 성장 결과를 확인할 수 있음

---

## 5. 데이터 구조

### CharacterData (ScriptableObject)

- 기본 스탯
- 성장 계수
- 필요 재료 정보

---

### CharacterRuntime

- 현재 레벨
- 현재 EXP
- 현재 스탯
- 승급 상태

---

### LevelingPolicy

성장 계산을 담당하는 정책 클래스

#### 역할
- EXP → 레벨 변환
- 레벨 제한 체크
- 승급 조건 검사
- 스탯 증가 계산

👉 성장 로직을 한 곳에 집중시켜 유지보수성을 확보

---

## 6. UI 구조

### Character Screen 구성

- 파티 캐릭터 선택
- 캐릭터 3D Preview (RenderTexture)
- 스탯 표시
- EXP 바
- 재료 선택 영역
- LevelUp / Promotion 버튼

---

### 주요 UI 특징

- 선택 즉시 프리뷰 반영
- 재료 수량 변경 시 실시간 계산
- 성장 결과를 시각적으로 표현

---

## 7. 설계 의도

### 1) 정책 분리

성장 계산을 `LevelingPolicy`로 분리하여  
데이터와 로직의 결합도를 낮췄습니다.

---

### 2) 데이터 중심 구조

CharacterData와 Runtime을 분리하여  
확장성과 유지보수성을 확보했습니다.

---

### 3) UX 중심 설계

프리뷰 시스템을 통해  
플레이어가 직관적으로 성장 결과를 이해할 수 있도록 설계했습니다.

---

### 4) 확장 가능성

향후 다음 요소 확장이 가능합니다:

- 장비 시스템
- 스킬 강화
- 특성 트리
- 캐릭터 각성 시스템
